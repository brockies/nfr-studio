# PCI DSS (Ecommerce) Notes (Anonymized)

This is a lightweight reminder document intended as a knowledge base source, not legal advice.

## Common Ecommerce Requirements Patterns
- Minimise PCI scope by using hosted payment fields / redirect flows where possible.
- Never store raw PAN or sensitive authentication data.
- Encrypt cardholder data in transit and at rest if it must exist in-scope.
- Maintain vulnerability management, patching SLAs, and penetration testing cadence.
- Strong access control and auditing for systems that touch payment flows.
